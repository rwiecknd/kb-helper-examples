Requesting a Google HSI (Highly Sensitive Information) Shared drive

This article is technical in nature on purpose since the process to create a Shared drive requires approval and following a strict series of steps/requests to verify the need and to correctly create an HSI Shared drive. We request that you work with your local IT Support related to this request. 

By default, a Google Shared drive is a secure space to store content and any one can create a shared drive. However, content with highly sensitive information (HSI) can only be stored in an HSI Shared drive.

HSI Shared Drive: This shared drive prevents sharing content outside the university and outside the Shared drive team members. The request will be reviewed by the OIT’s Information Security and Compliance team (they are responsible for knowing where all HSI content is stored at the university). The review will take a few days and the form submitter will be contacted by the InfoSec team. 

The OIT administrators can also create a Department Use Only Shared drive. This type of Shared drive has the same setup as an HSI Shared drive, but there is no review by OIT’s Information Security and Compliance team and the Shared drive will be available for use in a few days after making the request (see process below). A Departmental Use Only Shared drive is requested when an area/institute/center/department wants to treat their content similar to HSI. Please note that the restrictions are the same as an HSI Shared drive and there are no exceptions. Please only request a Department Use Only Shared drive when needed. And finally, as a reminder, HSI content cannot be stored in a Department Use Only shared drive.  

Contact your IT Support for assistance with HSI / Department Use Only Shared drives.

Before requesting a Secured Shared Drive: HSI or Sensitive content, as defined by the university, you need to:

  • Notify the Data Steward for your area of responsibility that you will be storing HSI/Sensitive data in a Google Shared drive
  • Obtain approval from the Data Steward who is responsible for the data before putting content in a Shared drive
  • Update both Data Stewards with additions/removal of people from the Google Group
  •  

To find the responsible Data Steward:

  • Go to data.nd.edu
  • Click Terms
  • In the search bar, search for an item. (ie; grades)
  • From the list returned, click on the item you plan to use/store
  • See any warning on the right
  • Click Governance tab

NOTE: If this is the first time accessing dataND, you will need to complete a brief quiz on Notre Dame’s Responsible Use of Data & IT Resources Policy. The quiz takes about 10 minutes to complete.

To Request a Secure Shared Drive

There are 2 steps for creating these particular types of Shared drives and need to be completed in the following order:

  1. Create a new AD group (see instructions below)
  2. Request an HSI or Department Use Only Shared drive (see instructions below)

__________

Step 1 - Create a New AD Group

Use this form to create a new AD group for your HSI data. Make sure you choose - AD and Google for HSI - as the Group Type. 

IMPORTANT: The Google Group name created from this form is needed to complete Step 2 to request an HSI or Department Use Only Shared drive. Once the AD Group name is completed move to Step 2 (Request an HSI Shared drive). 

More things to know:

  • We recommend that you create a new AD group for managing your HSI Shared drive rather than renaming an existing AD group
    • If you are using an AD group for access to another service, and prefer to rename an existing AD group, we can rename the group on an exception basis. If renaming, only individual NetID’s can be added as members
  • The AD group will end in “-g”, the form will enforce this standard, and that naming convention will automatically create the Google Group 
  • The Google Group name will be the same name as the AD group name
  • Members of the group can see the group through Google Groups. However, adding/removing members to this group is done only through groups.nd.edu, not in Google Groups
    • Group membership takes time to propagate - up to 6 hours
  • For security reasons, you cannot add shared/departmental accounts to groups that have access to HSI data

Step 2 - Request an HSI or Department Use Only Shared drive

Use this form to request an HSI  or Department Use Only Shared drive. 

  • Create a name for your Shared drive
    • For HSI Shared drives, we will add HSI to the beginning of the name
    • For Department Use Only Shared drives, we will add DeptUseOnly to the beginning of the name
    • Note - per a change early on in creating these types of Shared drives, we no longer want/require the underscore as part of the Shared drive name

Why use HSI or Department Use Only Shared drive? 

  • Settings are restricted in the Shared drive to prevent sharing content outside the university and outside the Shared drive team members
  • Limitations are placed on who manages the access to these settings and who is granted access

Other things to know about HSI or Department Use Only Shared drive:

  • Once the AD-Google Group is created, you manage access in https://groups.nd.edu through the Manage Access Group Membership form
  • Settings are in place to prevent sharing outside ND and to anyone who is not a Shared drive member
  • The Shared drive is created by the request form via ND’s Google administrator
  • Settings that are disabled with HSI and Department Use Only:
    • Restrict the ability to share any content, even a single file, with anyone outside the university
    • Restrict the ability to share any content, even a single file, with anyone who is not a member of the group. This setting limits who manages access and who is granted access
  • There is no exception to wanting to share even one file outside the standards noted above.
  • When the Shared drive is created, no one is granted access as a Manager
  • By not granting Manager access, the Shared drive settings cannot be changed nor the ability to change who has access to the Shared drive
  • Access is controlled by the AD-Google group within https://groups.nd.edu
  • Find more information on HSI and Sensitive data here.  
  • Data Steward engagement is imperative - Review Before requesting a Secured Shared Drive in the first section
  • Contact your IT Support for assistance with HSI / Department Use Only Shared drive

Who gets access to the Shared drive and what access is each person granted?

  • The Google Group itself is granted Content Manager access level when the Shared drive is created, which in turn gives all members in the group this access level
  • With Content Manager access, all Google Group members can add/edit/delete content as well as use the Google Drive for Desktop app (aka Drive File Stream)
  • To add or remove members go to https://groups.nd.edu through the Manage Access Group Membership form
  • If the Shared Drive contains HSI or Sensitive information, access to the drive must be approved by the appropriate data steward(s)

A: Yes, though it is not required. You only need to complete the first form from above, the Step 1 Service Now form, to create the AD-Google group.

A: Yes
