Overview

Requesting a Google HSI (Highly Sensitive Information) Shared drive

This article is technical in nature on purpose since the process to create a Shared drive requires approval and following a strict series of steps/requests to verify the need and to correctly create an HSI Shared drive. We request that you work with your local IT Support related to this request.

By default, a Google Shared drive is a secure space to store content and anyone can create a shared drive. However, content with highly sensitive information (HSI) can only be stored in an HSI Shared drive.

HSI Shared Drive: This shared drive prevents sharing content outside the university and outside the Shared drive team members. The request will be reviewed by the OIT’s Information Security and Compliance team (they are responsible for knowing where all HSI content is stored at the university). The review will take a few days and the form submitter will be contacted by the InfoSec team.

The OIT administrators can also create a Department Use Only Shared drive. This type of Shared drive has the same setup as an HSI Shared drive, but there is no review by OIT’s Information Security and Compliance team and the Shared drive will be available for use in a few days after making the request.

A Department Use Only Shared drive is requested when an area/institute/center/department wants to treat their content similar to HSI. Please note that the restrictions are the same as an HSI Shared drive and there are no exceptions. Please only request a Department Use Only Shared drive when needed. HSI content cannot be stored in a Department Use Only shared drive.

Contact Your IT Support

Contact your IT Support for assistance with HSI / Department Use Only Shared drives.

Before requesting a Secured Shared Drive: HSI or Sensitive content, as defined by the university, you need to:

  • Notify the Data Steward for your area that you will be storing HSI/Sensitive data in a Shared drive.
  • Obtain approval from the Data Steward before putting content in a Shared drive.
  • Update both Data Stewards with additions/removal of people from the Google Group.

To find the responsible Data Steward:

  • Go to data.nd.edu
  • Click Terms
  • Search for an item (e.g., grades)
  • Select the item
  • See the warning on the right
  • Click Governance tab

NOTE: First-time users must complete a quiz on Notre Dame’s Responsible Use of Data & IT Resources Policy (takes ~10 minutes).

To Request a Secure Shared Drive

There are 2 steps to create HSI or Department Use Only Shared drives:

  1. Create a new AD group
  2. Request a secure Shared drive

Step 1 - Create a New AD Group

Use this form to create a new AD group. Choose AD and Google for HSI as the Group Type.

IMPORTANT: The Google Group created here is required for Step 2. Once complete, proceed to request the Shared drive.

More things to know:

  • We recommend creating a new AD group rather than renaming an existing one.
    • If renaming an existing group, only individual NetIDs can be added.
  • The AD group name will end in “-g”; this name will also be used for the Google Group.
  • Group membership is managed via groups.nd.edu.
  • Changes can take up to 6 hours to propagate.
  • For security, shared/departmental accounts cannot be added to HSI-access groups.

Step 2 - Request an HSI or Department Use Only Shared drive

Use this form to request one.

  • Create a name for your Shared drive
    • HSI Shared drives will have "HSI" added to the beginning.
    • Department Use Only Shared drives will have "DeptUseOnly" added.
    • Do not use underscores in the name.

  • Settings prevent sharing content outside the university and outside team members.

Other Things to Know

  • AD-Google Group access is managed in https://groups.nd.edu.
  • Sharing outside of Notre Dame or to non-members is restricted.
  • Shared drives are created by the ND Google administrator via the request form.
  • Disabled settings include:
    • Sharing with anyone outside the university.
    • Sharing with non-members of the group.
  • No exceptions allowed for sharing outside defined boundaries.
  • No user is assigned as Manager upon drive creation.
  • Only the AD-Google Group controls access.
  • More info about HSI data is available here.
  • Review the section "Before requesting a Secured Shared Drive".
  • Contact IT Support for help.

Who Gets Access and What Access is Granted?

  • The Google Group is granted "Content Manager" access at time of drive creation.
  • All members can add/edit/delete files, and use Google Drive for Desktop.
  • Manage group membership at https://groups.nd.edu.
  • For drives with HSI/Sensitive data, Data Steward approval is required for access.

FAQs

Q: Can I manage a normal Shared drive via groups.nd.edu?
A: Yes, though it's not required. You only need to complete the Step 1 form to create the AD-Google group.

Q: If I don’t create an HSI or Department Use Only Shared drive, is my content secure in Google Drive?
A: Yes.
